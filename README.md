# Screenshot Tool

A lightweight screen capture tool for Linux and Windows built with Java 21 and JavaFX. Lives in the system tray and captures screen areas with global keyboard shortcuts.

## Features

- **System tray** integration вЂ” runs silently in the background
- **Two global hotkeys:**
  - `Ctrl+Alt+S` в†’ area selection capture
  - `Ctrl+Alt+W` в†’ active window capture
- **Area selection** with visual overlay, blue border, corner markers and вњ• cancel button
- **Active window capture** вЂ” instantly captures the focused window without manual selection
- **Multi-monitor** support вЂ” covers all screens simultaneously with screen labels
- **Single JVM instance** вЂ” trigger file system prevents multiple processes accumulating in memory
- **Native capture backends** with automatic fallback:
  - `gnome-screenshot` в†’ GNOME/Wayland/VirtualBox with Guest Additions
  - `scrot` в†’ X11, works in VirtualBox without Guest Additions
  - `xwd` в†’ X11 alternative, also works in VirtualBox
  - `import` (ImageMagick) в†’ X11 alternative
  - `Robot` (Java AWT) в†’ fallback for Windows and any platform
- **Auto-copy** to clipboard immediately after capture
- **HiDPI support** вЂ” correct resolution on high DPI screens (125%, 150%, 200%)
- **Save dialog** with:
  - Live preview of the capture
  - Filename input with timestamp default (`captura_2026-05-24_13-04-01`)
  - Format selector: `png`, `jpg`, `bmp`, `gif`
  - Folder selector (remembers last used, defaults to `~/Pictures`)
  - Toast notification after saving with "Open file" and "Show in folder" buttons (auto-closes in 4s)
  - Internal keyboard shortcuts: `Ctrl+S` save, `Ctrl+C` copy, `ESC` close, `Delete/F2` clear filename
- **Dark/light mode** вЂ” automatically adapts to system theme (Windows registry, Linux gsettings)

## Why jpackage instead of a shell/bat installer?

The original `install.sh` script compiled the project and installed the JAR, but had a critical problem: **JavaFX cannot be loaded correctly from a fat JAR** (generated by `maven-shade-plugin`) when specified via `--module-path` at runtime. The system `openjfx` package on Debian is version 11, incompatible with our JavaFX 21 dependency, causing:

```
Error: JavaFX runtime components are missing
Graphics Device initialization failed for: es2, sw
No suitable pipeline found
```

**`jpackage` solves this** by embedding the JRE and JavaFX 21 natively inside the installer package вЂ” no system Java or JavaFX required. The result is a self-contained app that works out of the box:

| | `install.sh` | `jpackage` |
|---|---|---|
| Requires Java on system | вњ… yes | вќЊ no (embedded) |
| Requires JavaFX on system | вњ… yes | вќЊ no (embedded) |
| JavaFX version conflict | вљ пёЏ possible | вњ… never |
| Installer size | ~5 MB | ~80 MB |
| Works offline | вњ… | вњ… |
| Appears in package manager | вќЊ | вњ… |
| Clean uninstall | manual | вњ… automatic |

The same approach is used for Windows with `.msi` + WiX Toolset.

## Installation (Linux)

```bash
git clone https://github.com/Francisco-Back/screenshot-tool.git
cd screenshot-tool
sudo dpkg -i installers/linux/screenshot-tool_2.0.0_amd64.deb
```

The `.deb` automatically:
- Installs the app with JRE and JavaFX 21 embedded
- Configures `Ctrl+Alt+S` and `Ctrl+Alt+W` hotkeys in GNOME
- Sets `StartupWMClass` so the correct icon appears in taskbar and dock
- Launches the app in the system tray

### Uninstall (Linux)

```bash
sudo dpkg -r screenshot-tool
```

Automatically removes GNOME hotkeys and stops the running process.

## Installation (Windows)

```batch
installers\windows\screenshot-tool-2.0.0.msi
```

The `.msi` automatically:
- Installs the app with JRE and JavaFX 21 embedded
- Configures `Ctrl+Alt+S` and `Ctrl+Alt+W` via Start Menu shortcuts
- Configures autostart on login
- Creates global `screenshot` command
- Appears in Control Panel в†’ Programs в†’ Screenshot Tool

### Uninstall (Windows)

Via Control Panel в†’ Programs в†’ Screenshot Tool в†’ Uninstall

Or via command line:
```batch
msiexec /x screenshot-tool-2.0.0.msi
```

Automatically removes hotkeys, autostart and `screenshot` command.

## VirtualBox

If running inside VirtualBox and the capture appears black, install Guest Additions for best quality:

```bash
sudo apt install virtualbox-guest-x11 virtualbox-guest-utils
sudo reboot
```

Without Guest Additions, `scrot` or `xwd` will be used automatically as fallback backends via X11.

## How the trigger system works

The trigger file system ensures only one JVM instance runs at a time:

**Linux:**
```
Ctrl+Alt+S (1st press)  в†’ app not running в†’ launches JVM в†’ stays in tray
Ctrl+Alt+S (2nd press)  в†’ app running в†’ touch /tmp/screenshottool.trigger
                        в†’ AppContext detects within 300ms в†’ area capture

Ctrl+Alt+W              в†’ touch /tmp/screenshottool.trigger.window
                        в†’ AppContext detects within 300ms в†’ window capture
```

**Windows:**
```
Ctrl+Alt+S в†’ HotkeyManager intercepts в†’ calls iniciarCaptura() directly
Ctrl+Alt+W в†’ HotkeyManager intercepts в†’ calls iniciarCapturaVentana() directly
```

No new JVM is ever created after the first launch вЂ” memory stays constant.
